## What is SOC 2?

SOC 2 (Service and Organisation Controls 2) is an auditing standard
developed by the AICPA. It evaluates whether a service organisation's
controls meet the Trust Service Criteria (TSC).

**SOC 2 Type I** — Tests whether controls are suitably designed
at a point in time.

**SOC 2 Type II** — Tests whether controls are operating effectively
over a period of time (typically 6-12 months). This is what enterprise
clients require.

### Trust Service Criteria

| Criteria | Abbreviation | Description |
|----------|-------------|-------------|
| Security | CC (Common Criteria) | Protection against unauthorised access |
| Availability | A | System available as committed |
| Processing Integrity | PI | Processing is complete, accurate, and authorised |
| Confidentiality | C | Confidential information protected |
| Privacy | P | Personal information collected and used appropriately |

CloudSafe is pursuing SOC 2 Type II covering all five criteria.

## Audit Readiness Assessment

**Overall Readiness: APPROACHING READY**

CloudSafe has a solid foundation of security controls. The most
significant gaps are in documentation and formal processes rather
than technical controls.

### Ready for audit
- Logical access controls (MFA, RBAC)
- Change management process
- Code review and deployment controls
- Privacy notice and consent
- NDA and confidentiality agreements

### Gaps to resolve before audit starts
- Formalise the risk assessment process and document results
- Write a data classification policy
- Document the data deletion and disposal process
- Write a capacity planning process
- Test backup restoration and document results
- Write a privacy incident procedure

---

## Recommended Audit Preparation Timeline

| Phase | Timeline | Activities |
|-------|----------|-----------|
| Remediation | Months 1-3 | Fix all 9 gaps. Document all controls formally. |
| Evidence collection | Months 4-9 | Collect 6 months of evidence showing controls operating |
| Pre-audit review | Month 10 | Internal review of all evidence with external auditor |
| SOC 2 Type II audit | Months 11-12 | Formal audit conducted by CPA firm |
| Report issued | Month 12 | SOC 2 Type II report delivered to CloudSafe |

---

## Evidence Collection Plan

For each control, CloudSafe must collect evidence showing it operated
consistently over the 6-month audit period.

| Control Type | Evidence Required | Frequency |
|-------------|------------------|-----------|
| MFA enforcement | Screenshot of MFA policy settings | Monthly screenshots |
| Access reviews | Completed access review records | Quarterly reviews |
| Vulnerability scanning | Scan reports with remediation evidence | Monthly scans |
| Incident response | Incident log with all incidents recorded | Ongoing |
| Change management | Change request records for all deployments | Per deployment |
| Backup testing | Restoration test reports | Quarterly tests |
| Security training | Training completion records | Annual + new hires |
| Risk assessment | Documented risk assessment | Annual |
